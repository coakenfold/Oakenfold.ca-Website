Single Page Application

Lead developer working on Fyndit.com. Split out dependencies by component. Used "js only" css classes as hooks for scripting: initializing, setting state, event binding. Delayed loading of 3rd party libraries until they were on the page or had been interacted with by the user. Used a variation of the App Shell model where new content is xhr-ed into a dedicated div. The site was setup to cache static elements of the page using redis and an inline .json object was used to "rehydrate" stateful interface elements (e.g: the user has liked this item, so change the background color) Routed all xhr requests through a script so that functons could be run pre/post request. Created a system where components register their js-hooks with a function. After an xhr update the function scans a portion of the dom for any existing classes and notifies the components that they exist. Using svg symbols for reuse.

Web Performance

Different strategies are required for the type of site that is being delivered. Strategic bundling of .js is a good idea for a single page application but it is not as relevant on a single page hand coded html/css site.

Successful strategies I have used in production:

  • require.js dynamic includes
  • lazy loading 3rd party
  • http/2
  • webpagetest
  • Cache static content, reduce expensive db calls, inline cache .json to rehydrate state

For the second release (V2) of fyndit.com, page load performance was tested at webpagetest.org against the initial load on a slow 3G connection using a mid-tier mobile device.

This latest version of the site outperformed the previous release in several key metrics:

  • First Interactive time went from 17.959s down to 7.918s. A 56% reduction.
  • Fully Loaded Bytes In size went from 1084kb down to 749kb. A 30% reduction.
  • Fully Loaded Time went down from 27.476s to 25.09s. A 8.67% reduction.

V2 also outperformed sample pages from Amazon 1 , Reddit 2 and Twitter 3 .

  • Fyndit's Fully Loaded Time is 58.42% faster than Amazon.
  • Fyndit's Fully Loaded Time is 29.79% faster than Reddit.
  • Fyndit's Fully Loaded Time is 6.27% faster than Twitter.
